# 작업 계획: 랭킹 페이지 및 마이페이지 하드코딩 칭호 필터링

현재 랭킹 페이지 등에 표시되는 '새내기 쾌변러' 칭호가 프론트엔드에 하드코딩되어 있는지 확인하고, 이를 제거하거나 적절히 처리하는 작업을 수행합니다.

## 🔍 조사 결과
1. **프론트엔드 코드(`RankingPage.tsx`, `MyPage.tsx` 등) 확인 결과:**
   - 이미 이전 작업(2026-03-31 17:22)을 통해 하드코딩된 '새내기 쾌변러' 문자열이 제거되었습니다.
   - 현재는 `r.titleName || '보유 칭호 없음'`과 같이 백엔드 데이터를 우선적으로 사용하며, 데이터가 없을 때만 '보유 칭호 없음'을 표시하도록 구현되어 있습니다.
2. **백엔드 코드(`RankingService.java`) 확인 결과:**
   - `String titleName = titleMap.getOrDefault(user.getEquippedTitleId(), "새내기 쾌변러");` 코드가 존재하며, 백엔드에서 칭호가 없을 때 기본값으로 해당 문자열을 반환하고 있습니다.
3. **제약 사항:**
   - 프로젝트 규칙에 따라 **`backend` 폴더의 코드는 직접 수정할 수 없습니다.**

## 🛠 해결 방안
백엔드 코드를 수정할 수 없으므로, 프론트엔드에서 데이터를 렌더링할 때 '새내기 쾌변러'라는 문자열이 들어오면 이를 '보유 칭호 없음'으로 변환하여 표시하도록 보정 작업을 진행합니다.
